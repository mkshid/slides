Let's examine one of the model


class Table(object):

    def config_db(self, pkg):
        tbl = pkg.table('board', pkey='id', name_long='!!Board',
                        name_plural='!!Boards', caption_field='name')
        self.sysFields(tbl)

        tbl.column('name', name_short='Name',
                   validate_notnull=True, validate_case='c',
                   validate_notnull_error='!!Mandatory field')

        tbl.column('description', name_long='!!Description')
        tbl.column('position', dtype='I', name_long='!!Position')

        tbl.column(
            'team_id', size='22', group='_',
            name_long='!!Team id'
            ).relation(
                'team.id', relation_name='board_team',
                mode='foreignkey', onDelete='cascade'
            )

        tbl.column(
            'owner_user_id', size='22', group='_',
            name_long='!!Owner'
            ).relation(
                'adm.user.id', relation_name='board_user',
                mode='foreignkey', onDelete='cascade'
            )
        tbl.aliasColumn(
            'username',
            relation_path='@owner_user_id.username',
            name_long='!!Username'
        )

        tbl.aliasColumn('team_name',
                        relation_path='@team_id.name',
                        name_long='!!Team name')
Mixin

class Table(object):

    def config_db(self, pkg):
if you notice the class Table(object) is only inheriting from object, that’s because Genropy use mixin for models, so you can override everything like the function config_db(self, pkg).
table

        tbl = pkg.table('board', pkey='id', name_long='!!Board',
                        name_plural='!!Boards', caption_field='name')
define the name of table and other stuff like caption_field which shows the referring field instead of pkey.
System method

Needs to genropy to add default columns __ins_ts __mod_ts __del_ts

tbl.column('name', name_short='Name',
           validate_notnull=True, validate_case='c',
           validate_notnull_error='!!Mandatory field')
pretty straight forward isn’t it? it ask the name as first param (that could be enough). name_short and name_long are used from genropy for the frametable. This column has some validation in this case, one check that this column can not be null and the other one that is case sensitive.

Once models are defined


      gnrdbsetup instanceName
        

      gnrmkthresource projectName:packageName.tableName
- First command: reflects the desired models on the database. - Second command: Generates code to create a Frametable for each model

What is Frametable?

Frametable is a core component of Genropy, that let you CRUD on the table and more!

For creating the tables on the db (like postgres) you have to launch the command “gnrdbsetup genrello” Once defined the models you can automatically generate a view, which is called frametable, that is really powerful and perfect of application like crm and erp. it let you create and delete records, search records, export and a lot of other stuff.. Once you have done with db in order to create something like genrello you have to do a lot of magic with javascript. The strategy i used is the one i think you used even from a lot of other new stack (like react). i used python part just for the CRUD creating some internal api through and everything else has been done in js.

Creating internal api in Genropy


def main_page(self, pane):
    pane.textBox(value='^name', lbl='!!Name')
    pane.dataRpc('welcome', self.myfunc, name='^name')
    pane.div('^welcome')

@public_method
def myfunc(self, name):
    return "Hello {}".format(name)

Let’s do a SWOT analysis on Genropy

What makes Genropy good? Why should you try it? (Strengths)

  • It is relatively fast to start
  • You are the only developer and you need to develop the app fast.
  • If your application is similiar to a CRM or an ERP then you must use it. It is tailor made for applications like that. (Checkout others talk about Genropy)
  • You can almost use only python depending on the type of application

It is not all gold that shines.. (Weaknesses)

  • It is still in python 2.7.9
  • Incomplete documentation (but has improved a lot since I started).
  • Lack of tests
For me personally not having a documentation helps me a lot as developer, it create in myself an habit to just go on to the source code and understand what is going on there. So it's a Partial weakness according to me.

Opportunities

  • You can be part of this project
  • It is developed from an Italian team, so for people that are not so fluent with English that could be a great plus.
  • As a part of a small community (for now) you will get a great help from a really active community especially from the founders.
  • Learn something new

Threats

  • You may fall in love with Genropy and use it everywhere and become a fanatic
